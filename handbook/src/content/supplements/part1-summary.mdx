## Chapter summaries

### Ch 1 — Principles & Mental Model

**The mental model**: Claude Code is an agent loop with finite context, observable tool use, and configurable behavior. Four engineering principles — never fail silently, simplicity, immutability, fail fast — govern every recommendation in the handbook. The codebase Claude reads becomes the curriculum it learns from.

- [Chapter 1: Principles & Mental Model](/chapters/ch01-principles) — the canonical reference (exercises + citations); its **Go deeper** block links the tutorial, TL;DR, how-to, and cheat sheet.

### Ch 2 — Your First CLAUDE.md

**The first permanent layer**: A starter CLAUDE.md captures what Claude cannot infer — build and test commands, architecture, verification criteria — and excludes what it can read from the codebase. Deny rules in `settings.json` protect secrets and destructive paths from day one. CLAUDE.md loads hierarchically (user → project → directory), specific overriding general. **Outcome**: a committed CLAUDE.md that makes your conventions stick across every session.

- [Chapter 2: Your First CLAUDE.md](/chapters/ch02-first-claude-md) — the canonical reference (exercises + citations); its **Go deeper** block links the tutorial, TL;DR, how-to, and cheat sheet.

### Ch 3 — Your First Working Session

**Running the loop reliably**: The session loop — prompt → observe → refine → commit — becomes repeatable once you add three prompting basics (specificity, step-by-step, a verification bar), context hygiene (`/clear`, `/compact`, the two-failure rule), Plan Mode for risky changes, and course correction (interrupt with Esc, back out with `/rewind`). **Outcome**: a session that finishes its task without drifting or running out of context.

- [Chapter 3: Your First Working Session](/chapters/ch03-first-session) — the canonical reference (exercises + citations); its **Go deeper** block links the tutorial, TL;DR, how-to, and cheat sheet.

### Ch 4 — Working Productively

**Talking to Claude precisely**: Precision vocabulary (replace vague verbs with checkable operations) and XML-tagged multi-component prompts (context / task / constraints / verification) make prompts land the first time more often. Scope control — naming the files Claude may touch, separating discovery from execution — keeps work from sprawling. Model selection matches model strength to task difficulty; extended thinking, caching, and cost optimization live in Appendix B. **Outcome**: prompts that work first-try more often than not.

- [Chapter 4: Working Productively](/chapters/ch04-productive) — the canonical reference (exercises + citations); its **Go deeper** block links the tutorial, TL;DR, how-to, and cheat sheet.

## What you'll have built by end of Part I

- A working **CLAUDE.md** in your repo
- A **`.claude/settings.json`** with reasonable permissions + deny rules
- A repeatable **session habit** (prompt → observe → refine, with `/clear` between unrelated tasks)
- A **prompting vocabulary** that produces consistent first-try results

That's enough to make Claude Code reliable. Parts II–IV build on this foundation — context management, agents, scaling configuration, team patterns, enterprise deployment.
