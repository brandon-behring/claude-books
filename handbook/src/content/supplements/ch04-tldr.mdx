<KeyIdea>
Inconsistent results are a **precision** problem, not luck. Replace vague verbs with **operations**, structure complex requests with **XML tags** (`context` / `task` / `constraints` / `verification`), stop **scope creep** by naming the files Claude may touch, and **match model strength to task difficulty**.
</KeyIdea>

## A precision vocabulary

Vague verbs are where Claude guesses. Name the operations instead:

| Vague | Precise |
|---|---|
| "Clean this data" | validate schema, impute nulls with median, drop `NaN`-target rows |
| "Check if this works" | run `pytest`, confirm no leakage, verify distributions match prod |

Longer, but every clause is checkable. **Compounding payoff**: when your `CLAUDE.md` and your prompts use the same precise verbs, correction rates fall.

## XML tags for complex prompts

For a request with **three or more parts**, separate them so Claude does not have to:

```text
<context>      what Claude can't infer (state, locations, the metric now)
<task>         the single change you want
<constraints>  boundaries to respect WHILE working (in-bounds files, interfaces)
<verification> how success is checked AFTER (a command, a behaviour)
```

The discipline of filling four sections is the win, not the brackets. **Rich inputs**: hand Claude the artifact (`@file`, pasted screenshots, piped logs), not a description of it.

## Scope control

Scope creep is a **prompting gap, not a model fault** — Claude touched the extra file because the prompt did not forbid it. Three fixes:

```text
1. Name the files       "modify session.py + its test, nothing else"  (simplest)
2. Discovery → action   "read src/payments/ and tell me what changes — no edits yet"
3. Plan Mode            the read-only mode enforces it with permissions, not words
```

When Claude strays, add a constraint to the next prompt — don't just undo it.

## Match the model to the task

```text
Faster / lighter  → mechanical, well-specified (formatting, boilerplate, bulk)
Balanced          → most development (the daily default)
Frontier          → ambiguous / complex (architecture, multi-file, unclear bugs)
```
