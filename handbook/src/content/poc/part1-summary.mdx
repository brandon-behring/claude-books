## Chapter summaries

### Ch 1 — Principles & Mental Model

**The mental model**: Claude Code is an agent loop with finite context, observable tool use, and configurable behavior. Four engineering principles (never fail silently, simplicity, immutability, fail fast) govern every recommendation in the handbook. The codebase Claude reads becomes the curriculum it learns from.

- Tutorial: full chapter with worked example + retrieval prompts (~15 min)
- TL;DR: compressed essence (~5 min)
- How-to: CLAUDE.md for a monorepo (~5 min)
- Cheat sheet: agent-loop diagram + principle reference card

### Ch 3 — Your First Working Session

*Placeholder summary — chapter not yet rewritten.*

**What it covers**: the repeatable session loop (prompt → observe → refine → commit). Three prompting basics (specificity, step-by-step, verification criteria). Context hygiene (`/clear`, `/compact`, the two-failure rule). Plan Mode for complex tasks. Course correction (Esc, `/rewind`).

**Outcome**: a session that finishes its task without drifting or running out of context.

**Outcome**: prompts that work the first time more often than not.

## What you'll have built by end of Part I

- A working **CLAUDE.md** in your repo
- A **`.claude/settings.json`** with reasonable permissions + deny rules
- A repeatable **session habit** (prompt → observe → refine, with `/clear` between unrelated tasks)
- A **prompting vocabulary** that produces consistent first-try results

That's enough to make Claude Code reliable. Parts II–IV build on this foundation — context management, agents, scaling configuration, team patterns, enterprise deployment.
