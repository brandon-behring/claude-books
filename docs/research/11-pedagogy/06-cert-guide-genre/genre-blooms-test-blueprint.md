# Test blueprint + Bloom's-aligned item writing — how the question bank is engineered

## Author + context

- **Source**: Baylor College of Medicine, *Principles and Guidelines for Assessments* (revised 6/15/15), incorporating a "Modified Bloom's Taxonomy for Evaluating Multiple Choice Questions" framework and a "Blueprint of Tests" section. An institutional assessment-design standard (the methodology layer beneath any cert question bank), drawing on Bloom (1984), Knecht (2001), and Palmer & Devitt (2007).
- Tier: an institutional guidelines document (not a certifying-body accreditation doc and not peer-reviewed primary research), so **T3-practitioner** — but it cites and operationalizes the measurement literature, so it is the most rigorous *item-writing* source in this set.
- This note is the **meta / "how to engineer the review apparatus"** source. The other four notes describe what cert books *look like*; this one says how to make the questions inside them *valid* — the table-of-specifications + Bloom's level + item-construction rules.

## Core principle / theory

Four engineering ideas matter for the Architect's Reference question apparatus:

1. **Lower-order vs. higher-order items are a deliberate mix, set by blueprint quota.** Lower-order items "require the learner to recall or comprehend learned information without application." Higher-order items "require the learner to use prior knowledge to apply new information … and analyze, synthesize and/or evaluate." A *test blueprint* sets the target *ratio* (e.g. "minimum of 40% higher-order items"). You don't write questions ad hoc; you fill a specification.
2. **The vignette + lead-in item structure.** A strong applied item = a **stem** (a *vignette*: a realistic scenario "that provides the examinee all the information needed to answer") + a **lead-in** (the final sentence posing the question), then a **key** (correct answer) and plausible **distractors**. This is exactly the "real-world scenario" item the cert genre prizes — formalized.
3. **The "cover test" quality gate.** "Can examinees pose an answer to the question based solely on the question stem?" If the stem only makes sense once you read the options, it's a weak item. Plus concrete prohibitions: no "all/none of the above," no K-type ("A and C") items, no negatively-worded items (unless the point is to test what you *don't* know).
4. **Every item carries author-written feedback.** "All questions should have meaningful 'feedback' provided by the question writer" — i.e. an answer *rationale* is mandatory, not optional. And items are revised out if they perform poorly (low discrimination / difficulty indices).

## Empirical backing

The document grounds its item categories in the measurement literature (Bloom's *Taxonomy of Educational Objectives*; Knecht 2001 on classifying pharmacy MCQs by Bloom's level; Palmer & Devitt 2007 on higher-order assessment). It also prescribes *post-hoc psychometrics*: discrimination index and difficulty index thresholds (flag items with difficulty ≤ 0.50 and discrimination < 0.30, or negative discrimination). So "good question" is operationalized statistically, not by taste. This is the assessment-design counterpart to the learning-science in [[theory-bjork-desirable-difficulties]] and the cognitive ladder in [[theory-blooms-taxonomy]].

## Application to a certification study guide

This converts the cert genre's "review questions with rationales" and "real-world scenario" conventions from vibes into a spec the Architect's Reference can actually follow:

- **Set a Bloom's ratio for the question apparatus.** Don't let chapter Practice items default to all-recall. Target a mix — e.g. the chapter self-check leans recall/comprehension (router/diagnostic role), while the pooled practice exam leans higher-order (scenario application). This formalizes the two-tier split the Pearson/Sybex notes describe.
- **Write applied items as vignette + lead-in.** A "real-world scenario" Practice item should be a short realistic situation (the stem) ending in a single clear lead-in question — the genre's scenario callout, done as a valid item.
- **Apply the cover test + prohibitions** to every Practice item: the stem must be answerable before reading options; no all/none-of-the-above; no negative phrasing unless intentional. Cheap quality gates that raise item validity immediately.
- **Mandate a rationale per item.** The template already wants answer rationales; this source elevates that to a hard rule ("meaningful feedback … provided by the question writer") and is the citation for it.

## Concrete recommendations for claude-books

- **Define a target cognitive mix per tier** in the book's question-apparatus plan: chapter self-check = mostly Remember/Understand (fast diagnostic); pooled practice exam = mostly Apply/Analyze (scenario). Make the ratio explicit, blueprint-style, not incidental.
- **Standardize the scenario Exercise as vignette + single lead-in** (realistic situation → one clear question), so "real-world scenario" callouts are valid items, not decoration.
- **Adopt a 4-rule item-quality checklist**: (1) passes the cover test, (2) no all/none-of-the-above or K-type, (3) no negative wording unless deliberate, (4) ships with an author rationale. Put it in the authoring conventions for the review apparatus.
- **Make answer rationales mandatory** for every Practice/review item (not just a key) — this source is the citation backing that rule.
- **Plan a pruning pass**: if/when the book ever collects response data, retire items by difficulty/discrimination thresholds; until then, peer-review items against the checklist as a proxy.
