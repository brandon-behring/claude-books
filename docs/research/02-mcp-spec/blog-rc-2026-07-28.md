# MCP 2026-07-28 Release Candidate — breaking changes + migration guide

**RC locked May 21, 2026; final publication July 28, 2026.** This is the single most important MCP document for chapter authors writing right now — it tells you which 2025-11-25 mechanisms will be obsolete by the time the books ship. The headline shift is **stateless protocol core** (no more `initialize`/`initialized` handshake, no `Mcp-Session-Id`). Three features get deprecated with 12-month removal windows. The RC also introduces an **Extensions framework**, **MCP Apps** (server-rendered HTML in sandboxed iframes), required headers for routing, JSON Schema 2020-12 composition, and OAuth 2.1 tightening. Marked `volatility: fast-moving` — verify against the final spec on/after July 28, 2026.

## Key takeaways
- **Stateless protocol core (SEP-2575, SEP-2567)**: `initialize` / `notifications/initialized` handshake **removed**. `Mcp-Session-Id` header **removed**. Protocol-level sessions gone. Protocol metadata moves to `_meta` on every request.
- **New required headers (SEP-2243)**: `MCP-Protocol-Version: 2026-07-28`, `Mcp-Method: tools/call`, `Mcp-Name: search`. Servers **MUST** reject requests where headers and body disagree. Lets load balancers route on operation without body inspection.
- **`server/discover` method (new)** replaces the handshake for capability lookup.
- **Three features deprecated (12-month removal window, SEP-2577)**: **Roots**, **Sampling**, **Logging**. Annotation-only deprecation — methods, types, capability flags remain functional until at least May 2027.
  - **Roots** -> use tool parameters, resource URIs, or server configuration.
  - **Sampling** -> integrate LLM provider APIs directly from the server.
  - **Logging** -> use stderr (stdio) or OpenTelemetry (structured observability).
- **Tasks graduates to extension (SEP-2663)**: no longer in core. Lifecycle restructured around the stateless model. `tasks/list` **removed** (can't scope safely without sessions). Server now decides when a call runs as a task (not the client). 2025-11-25 experimental Tasks implementations **must migrate**.
- **Server-to-client requests** now only allowed during active client request processing (SEP-2260). Multi-round-trip requests replaced with `InputRequiredResult` payload pattern (SEP-2322). **SSE stream pattern for elicitation removed**.
- **Error code change (SEP-2164)**: resource-not-found shifts from proprietary `-32002` to JSON-RPC standard `-32602` (Invalid Params).
- **JSON Schema 2020-12 composition (SEP-2106)**: `inputSchema`/`outputSchema` now support `oneOf`, `anyOf`, `allOf`, conditionals, `$ref`, `$defs`. `inputSchema` keeps `type: "object"` root constraint; `outputSchema` is unrestricted. `structuredContent` accepts any JSON value. Security: **MUST NOT auto-dereference external `$ref` URIs**; **SHOULD** bound schema depth/validation time.
- **Authorization tightening (six SEPs)**:
  - **SEP-2468**: Clients **MUST** validate `iss` parameter on auth responses (RFC 9207, mix-up attack mitigation). Future version will reject responses omitting `iss`.
  - **SEP-837**: Clients **MUST** declare OpenID Connect `application_type` during DCR (prevents AS defaulting desktop/CLI clients to `"web"`).
  - **SEP-2352**: Registered credentials **MUST** be bound to the issuing AS's `issuer`; re-registration required when resource migrates between authorization servers.
  - **SEP-2207**: Documented refresh token request process.
  - **SEP-2350**: Step-up authorization scope accumulation behavior clarified.
  - **SEP-2351**: `.well-known` discovery endpoint suffix clarified.
- **Extensions framework (SEP-2133)**: reverse-DNS IDs, negotiated via `extensions` map. Extensions live in dedicated `ext-*` repositories with delegated maintainers, version independently. Two official extensions ship in this release: **Tasks** and **MCP Apps**.
- **MCP Apps (SEP-1865)**: servers ship interactive HTML interfaces in sandboxed iframes. UI templates declared upfront so hosts can prefetch, cache, security-review before execution. Every UI-initiated action flows through audit/consent as a tool call.
- **Caching support**: list and resource-read results carry `ttlMs` and `cacheScope` (modeled on HTTP `Cache-Control`).
- **W3C Trace Context propagation (SEP-414)**: `_meta` carries `traceparent`, `tracestate`, `baggage` for distributed tracing.
- **SDK tiering**: Standards Track SEP cannot reach Final until matching scenario lands in conformance suite (SEP-2484). Official SDKs scored against the new tier system. Tier 1 SDKs expected to ship 2026-07-28 support within the 10-week validation window.

## Migration summary: per-feature table

| 2025-11-25 mechanism | 2026-07-28 RC change | Replacement / Migration |
|---|---|---|
| `initialize` request + `notifications/initialized` | **Removed** | Stateless: capability info via `server/discover`. Protocol version travels in `MCP-Protocol-Version` header. |
| `Mcp-Session-Id` header | **Removed** | No protocol-level sessions. Stateful behavior is application-level. |
| SSE stream for elicitation | **Removed** | `InputRequiredResult` payload pattern (SEP-2322). |
| Server-to-client requests anytime | **Restricted** | Only allowed during active client request processing (SEP-2260). |
| `tasks/list` | **Removed** | No replacement — can't scope without sessions. |
| Tasks client-driven creation | **Server-driven** | Client advertises extension; server decides per call. |
| Tasks in core spec | **Extension** | Tasks moves out of core; lives in `ext-tasks`. |
| Resource not found `-32002` | **`-32602`** | Standard JSON-RPC Invalid Params (SEP-2164). |
| `inputSchema` limited composition | **Full 2020-12** | `oneOf` / `anyOf` / `allOf` / conditionals / `$ref` / `$defs` allowed. |
| External `$ref` auto-dereference | **Forbidden** | Security: implementations MUST NOT fetch external `$ref` URIs. |
| **Roots** (client capability) | **Deprecated (12mo)** | Tool params, resource URIs, or server config. |
| **Sampling** (client capability) | **Deprecated (12mo)** | Integrate LLM provider APIs directly from server. |
| **Logging** (server capability) | **Deprecated (12mo)** | stderr (stdio) or OpenTelemetry (HTTP). |
| OAuth: `iss` validation | **MUST** validate per RFC 9207 | SEP-2468. Future: reject responses missing `iss`. |
| OAuth: DCR | **MUST** declare `application_type` | SEP-837. Prevents desktop/CLI clients defaulting to `"web"`. |
| OAuth: credentials | **MUST** bind to AS `issuer` | SEP-2352. Re-register when resource migrates AS. |

## New: required headers on Streamable HTTP

Per SEP-2243, every HTTP request **MUST** carry:
```
MCP-Protocol-Version: 2026-07-28
Mcp-Method: tools/call
Mcp-Name: search
```

Servers **MUST** reject requests where headers and body disagree. Lets:
- Load balancers route by operation type without body inspection.
- Gateways apply rate limiting / auth policies per-method.
- Observability tools categorize traffic at the edge.

## New: `server/discover`

Replaces the `initialize` handshake for capability lookup. Clients can fetch server capabilities without establishing a session.

## New: Extensions framework (SEP-2133)

- **Naming**: reverse-DNS IDs (e.g., `io.modelcontextprotocol.tasks`, `io.modelcontextprotocol.apps`).
- **Negotiation**: via `extensions` map on client/server capabilities (parallels `capabilities` for core features).
- **Repos**: dedicated `ext-*` repositories with delegated maintainers.
- **Versioning**: independent of core spec.
- **SEP track**: experimental -> official. Standards Track SEP can't reach Final without matching conformance suite scenario (SEP-2484).
- **Officials**: Tasks (`ext-tasks`) and MCP Apps (`ext-apps`) ship as official extensions in this release.

## New: MCP Apps (SEP-1865)

- Servers ship interactive HTML interfaces rendered in **sandboxed iframes**.
- Tools declare UI templates ahead of time.
- Hosts can **prefetch**, **cache**, **security-review** templates before they execute.
- UI communicates over the same JSON-RPC channel as the rest of MCP.
- Every UI-initiated action flows through the audit / consent path as a direct tool call.

Practical implication: a server can ship an interactive Asana-board / Datadog-dashboard / Figma-canvas without the host needing a custom integration.

## New: caching semantics

Tool list, resource list, resource read results now carry `ttlMs` and `cacheScope` headers modeled on HTTP `Cache-Control`. Hosts can cache MCP responses with proper invalidation.

## New: W3C Trace Context (SEP-414)

`_meta` carries:
- `traceparent` (W3C trace ID + span ID)
- `tracestate` (vendor-specific extensions)
- `baggage` (cross-cutting context)

Lets MCP traffic participate in distributed traces alongside other services.

## How chapter authors should migrate content

1. **For 2025-11-25 chapters published before July 28, 2026**: teach the current spec as-is. Add an "evolving" callout at the top of any section touching session management, lifecycle, tasks, roots, sampling, or logging.
2. **For chapters published after July 28, 2026**: teach the 2026-07-28 model. Treat `initialize`/`Mcp-Session-Id` as historical. Keep Roots/Sampling/Logging in a "legacy" section noting the May 2027+ removal target.
3. **For any chapter covering OAuth**: cite the six SEPs above as the canonical "tighter auth" reference. The 2026-07-28 RC's auth rules are a strict superset of 2025-11-25 — code written for the RC will work on 2025-11-25, not vice versa.
